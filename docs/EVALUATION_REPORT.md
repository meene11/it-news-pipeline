## 3. 정량 평가 요약

### 3.1 감성 분류 정확도

| 지표 | 값 |
|------|-----|
| **전체 일치율** | **13/20 = 65%** |
| 정답 (일치) | 13건 |
| 오답 (불일치) | 7건 |

### 3.2 감성별 정확도

| AI 감성 | AI 판단 건수 | 일치 건수 | 정확도 |
|---------|-------------|-----------|--------|
| positive | 12건 | 6건 | **50%** |
| negative | 4건 | 4건 | **100%** |
| neutral | 4건 | 3건 (나머지 1건도 AI가 neutral로 정확) | **100%** |

### 3.3 오류 패턴 분석

| 오류 유형 | 건수 | 설명 |
|-----------|------|------|
| **긍정 과잉 판정** | 7건 | 중립적 사실 보도를 긍정으로 판단하는 경향 |
| 부정 과잉 판정 | 0건 | 해당 없음 |
| 중립 과잉 판정 | 0건 | 해당 없음 |

**핵심 발견: AI(gpt-4o-mini)는 "긍정 편향(Positive Bias)"을 보입니다.**

- '승리', '돌파', '등장', '발표' 같은 단어를 맥락과 무관하게 긍정으로 분류하는 경향
- 부정 판단은 매우 정확함 (100%)
- 중립 판단도 정확하지만, 중립이어야 할 것을 긍정으로 잡는 경우가 많음

### 3.4 Confidence Score 보정도 분석

| Confidence 구간 | 건수 | 일치 건수 | 실제 정확도 | 보정 평가 |
|-----------------|------|-----------|-------------|-----------|
| 0.90 | 3건 | 2건 | 67% | 과신 (90% 자신감인데 67% 정확) |
| 0.80 | 15건 | 10건 | 67% | 과신 (80% 자신감인데 67% 정확) |
| 0.70 | 2건 | 2건 | 100% | 과소평가 (겸손했는데 맞음) |

**Confidence Score는 실제 정확도를 반영하지 못합니다.**
- 대부분 0.80으로 몰려 있어 변별력이 낮음
- 높은 신뢰도(0.90)가 높은 정확도를 의미하지 않음
- 이것은 LLM의 self-reported confidence의 알려진 한계

### 3.5 요약 품질 평가

| 평가 항목 | 결과 |
|-----------|------|
| 핵심 내용 포함 | 20/20 (100%) |
| 20자 이내 준수 | 18/20 (90%) |
| 오해 소지 없음 | 19/20 (95%) |
| 전반적 요약 품질 | **양호** |

요약은 감성 분석보다 품질이 높습니다. 제목의 핵심을 잘 추출합니다.

## 4. 경향성 탐색 기능의 한계 분석

### 4.1 현재 데이터의 구조적 한계

| 한계 | 현재 상태 | 심각도 |
|------|-----------|--------|
| **단일 매체** | ETNews 1곳에서만 수집됨 | 치명적 |
| **표본 크기** | 20건 | 높음 |
| **매체 간 비교 불가** | 비교 대상 매체가 없음 | 치명적 |
| **기간** | 1일치 데이터 | 높음 |

현재 데이터로는 **매체 간 경향성 비교가 불가능합니다.** Bias Score를 산출할 수 있는 최소 조건(2개 이상 매체, 동일 키워드)을 충족하지 못합니다.

### 4.2 감성 분석 자체의 한계

| 한계 | 설명 | 영향 |
|------|------|------|
| **긍정 편향** | AI가 중립을 긍정으로 판단하는 경향 | 경향성 분석 결과가 긍정 쪽으로 왜곡 |
| **제목만 분석** | 본문 없이 제목만으로 판단 | 낚시성 제목에 취약 |
| **Confidence 무의미** | 대부분 0.8로 몰려 변별력 없음 | 자동 품질 관리 불가 |
| **한국어 뉘앙스** | '승리', '돌파' 등을 맥락 없이 긍정 처리 | 정치/사회 뉴스에서 오류율 높음 |

### 4.3 정직한 결론

이 프로젝트의 AI 감성 분석은:
- **요약 기능**: 양호 (95% 이상 품질)
- **감성 분류**: 불충분 (65% 정확도, 산업 기준 80% 미달)
- **경향성 탐색**: 현재 데이터로는 의미 있는 분석 불가

**면접에서 이렇게 말할 수 있습니다:**
> "직접 20건 전수 평가한 결과 감성 분류 정확도는 65%였습니다. 
> 특히 AI가 중립을 긍정으로 잡는 '긍정 편향' 패턴을 발견했고,
> 이를 개선하려면 프롬프트 튜닝이나 few-shot 예시 추가가 필요합니다.
> 현재 데이터가 단일 매체 20건이라 경향성 비교는 불가하며,
> 최소 3개 매체에서 주 단위로 수집해야 의미 있는 분석이 가능합니다."

## 5. 개선 방안

### 5.1 감성 분류 정확도 개선

| 방안 | 예상 효과 | 난이도 |
|------|-----------|--------|
| **프롬프트에 few-shot 예시 추가** | 10~15% 향상 예상 | 낮음 |
| **"사실 보도는 neutral로 분류" 규칙 강화** | 긍정 과잉 감소 | 낮음 |
| **gpt-4o로 모델 업그레이드** | 5~10% 향상 예상 | 비용 증가 |
| **2단계 검증 (1차 분류 → 2차 확인)** | 15~20% 향상 예상 | 비용 2배 |

### 5.2 데이터 품질 개선

| 방안 | 설명 |
|------|------|
| RSS 소스 추가 | ZDNet, Bloter가 수집되도록 Fallback 순서 조정 |
| 수집 기간 확대 | GitHub Actions로 매일 수집하여 주/월 단위 데이터 확보 |
| 매체 식별 강화 | RSS URL 기반 자동 매체 태깅 |

### 5.3 Confidence Score 개선

| 방안 | 설명 |
|------|------|
| Temperature 조정 | 0.1 → 0.0으로 낮춰 일관성 향상 |
| 프롬프트 보정 | "확실하지 않으면 0.5 미만으로 설정" 지시 강화 |
| 외부 보정 | 감성 사전(KNU 감성 사전)과 LLM 결과를 비교하여 보정 |

## 6. 면접에서 꼭 알아야 할 것

### "AI 분석 정확도가 65%인데 괜찮은 건가요?"

> 65%는 산업 기준(80%)에 미달하지만, 이 프로젝트의 목적은 높은 정확도를 달성하는 것이 아니라
> **AI 파이프라인을 구축하고, 품질을 정량적으로 평가하고, 개선 방향을 도출하는 과정**을 보여주는 것입니다.
> 실제로 평가를 통해 "긍정 편향" 패턴을 발견했고, 이를 개선하기 위한 구체적 방안(few-shot, 프롬프트 튜닝)을 제시할 수 있습니다.

### "왜 편향 분석이 아니라 경향성 탐색인가요?"

> "편향 분석"은 특정 매체가 편향되었다고 판정하는 것을 의미하는데,
> 그러려면 대규모 데이터, 복수 평가자, 통계적 유의성 검정이 필요합니다.
> 이 프로젝트는 LLM 감성 분석 기반으로 매체 간 보도 톤 차이의 경향성을 탐색하는 수준이며,
> 그 한계를 인지하고 정직하게 "경향성 탐색"이라고 명명했습니다.

### "LLM 감성 분석의 한계를 어떻게 보완할 수 있나요?"

> 1) 프롬프트에 few-shot 예시를 추가하여 "사실 보도는 neutral" 기준을 학습시킵니다.
> 2) 감성 사전(KNU 한국어 감성 사전)과 LLM 결과를 앙상블하여 교차 검증합니다.
> 3) Human evaluation을 주기적으로 수행하여 정확도 변화를 모니터링합니다.
> 4) Confidence Score 대신 로짓 기반 확률을 사용하거나, 복수 모델 합의를 확인합니다.
